FCAD Summer internship

This internship ran for 10 weeks from June to August at UMass. I'm working with Professor Gopal Narayanan and graduate student Sandra Bustamante to create a bias card enclosure to add to the OMAyA cryostat chamber. The mechanical design of the enclosure includes computer aided design through SOLIDWORKS, creating machine ready drawings, verifying the integrity of the components, and the assembly of the parts. The enclosure will screw into the top of the cryostat chamber with the purpose of enclosing and connecting the bias cards.

OMAyA stands for One Millimeter Array for Astronomy and will be installed in the Large Millimeter Telescope (LMT). It is an 8-pixel focal plane heterodyne receiver for the 1 mm band (210 to 280 GHz). It includes dual-polarization receivers and sideband separation mixers, creating extreme sensitivity. These mixers use SIS junctions which are considerably good heterodyne mixers because they have noise comparable to the quantum limit and a conversion gain that is less than 1, making them ideal for radio observations. OMAyA will be equipped with an On-The-Fly (OTF) mapping mode allowing it to create 10’ x 10’ maps with an RMS ~ 0.1K and an angular resolution of ~ 6 arcseconds in 80 minutes, implementing an innovative separation system that has a high level integration of amplifier, mixer, and local oscillator components.

Toltec Unity Cluster

For two semesters, I worked with Professor Robert Gutermuth using Alyssa Sokol’s paper from 2019, Early science with the Large Millimetre Telescope: An LMT/AzTEC 1.1 mm Survey of dense cores in the Monoceros R2 giant molecular cloud as well as Sarah Betti’s 2021 paper, Robustness of Synthetic Observations in Producing Observed Core Properties: Predictions for the TolTEC Clouds to Cores Legacy Survey as guides for my undergraduate research.

During my first semester, I used TolTEC data of the Ophiuchus molecular cloud and scaled it back to 1400 parsecs. After I did this, I ran many different simulations and reductions using the Unity cluster to eventually pass through Sarah Betti’s ImSeg code. This code was able to take my reduced data and pull out the number of star cores that were left. I unfortunately was left with many problems at the end of this semester including what we called an ‘artifact’ that kept appearing in my reduced images. This data was able to help the development team fix the problem with the simulated atmosphere before new TolTEC data got released.

The second semester with Professor Gutermuth was focused on using STARFORGE data to run simulations. Using the M2e3 simulated cloud data, I ran my simulations and reductions once again on the Unity cluster. As this work was becoming quite tedious and was taking up a lot of space on my local computer, I was able to create a python script with the help of my professor to upload to Unity that would quickly print out what the image looked like with added contours scaled at 2.5, 5, 10, and 20 on the noise and weight map. This “quick view” script also printed out the noise value as well as the footprint value. This allowed me to recognize if anything had gone wrong before I downloaded each individual file. Although there was some downtime on the cluster since it is a developing software, I feel as though my work was able to help the development team fix what needed to be changed as I was continuously using the software running simulations and overall made the tedious work easier with my script.
